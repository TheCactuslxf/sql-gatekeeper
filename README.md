# SQL Gatekeeper

Python implementation of a SQL gatekeeper service for LLM-generated MySQL SQL.

## Goals

- Accept SQL generated by an LLM.
- Route logical tables to physical tables across multiple MySQL instances.
- Rewrite SQL when the input only contains logical table names.
- Run automatic safety checks before execution.
- Reject high-risk SQL without human approval.
- Execute approved SQL with read-only credentials.

## Constraints

- Language: Python
- API style: service/API form
- Database target: MySQL
- Local development database: Docker MySQL
- Every coding task must include unit tests
- Safety checks are performed on rewritten physical SQL

## High-Level Modules

- API layer
- SQL parser
- Route decision engine
- SQL rewrite engine
- Security filter chain
- Metadata / EXPLAIN risk evaluator
- Executor
- Audit logger

## Initial Development Order

1. Project skeleton and local Docker environment
2. Metadata schema and repositories
3. API contracts
4. SQL parsing and route decision
5. SQL rewrite
6. Filter chain and first security filters
7. Metadata loading and EXPLAIN-based risk checks
8. Execution, audit, and end-to-end tests
